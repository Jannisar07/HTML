Q.9 How is Cell Padding different from Cell Spacing?

Answer:

Cellpadding

In HTML table layout, the cellpadding attribute is used to indicate the distance between a cell's edge and its content. Cellpadding, as its name suggests, is an attribute of the table tag , which specifies how far the cell border and content are separated from one another. HTML's cellpadding attribute takes values in percentages or pixels. This value may be 0 based on the user's requirements.

Code:

<table border="1" cellpadding="10" >

Cellspacing:

Cell spacing is one more attribute of the table tag in HTML. It controls the distance between two adjacent cells. Therefore we can easily change the space between adjacent cells. It mainly serves to improve readability of the table.

Code:

<table border="1" cellspacing="10" >

Note* The most fundamental distinction between cellspacing and cellpadding is that cellspacing is used to set the whitespace between two cells, and cellpadding is used to set the whitespace between a cell's edge and its content.

Q-11. What is the difference between a block-level element and an inline element?

Answer:

Block Element

A block-level element always begins on a new line, and browsers add a space before and after the element by default. A block-level element always occupies the entire available width and extends as far to the left and right as it can. Eg of block elements: <p > <Div >

Inline Element:

Inline components don't interfere with the content's flow; instead, they only take up the space defined by the tags specifying them.

Note* the basic difference is that Every block element begins on a new line. There is never a new line before an inline element. Block elements completely fill the available area from left to right. Only the area defined by the tags in the HTML element is covered by inline elements.
